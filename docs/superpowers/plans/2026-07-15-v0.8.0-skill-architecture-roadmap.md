# v0.8.0 Skill 架构重构路线图（小版本迭代计划）

**日期**: 2026-07-15
**基线**: v0.7.1-release（master 已推送）
**目标**: 以 skill 组为核心，让"用户需求 → 路径选择 → 引擎调用 → 报告交付"全程可控，为管线化打基础
**实施准则**: skill 设计最佳实践 —— 渐进式披露（SKILL.md 精简 + references/ 深挖）、单一事实源（阈值/权重只在 engine 文档，skill 不复制）、显式协议（协议写在明处）、机器可读（路径单结构化）

---

## 一、版本路线总览

| 版本 | 主题 | 核心交付物 | 测试焦点 |
|---|---|---|---|
| **v0.7.2** | 工作路径注册表（全量设计基线） | `dev/engine/work-path-registry.md` + 路径 schema | 注册表自身的完整性/一致性 |
| **v0.7.3** | Intake Router skill（需求理解层） | `credit-analysis-router` skill + 工作路径单 schema | 场景→路径推荐正确性 |

| **v0.7.5** | 管线化基础 | 路径单验证器 + L0→L1→L2 链式规则 | 验证器单测 + 链路一致性 |
| **v0.8.0** | 集成发布 | 全链路打通 + 版本晋升 + 快照 | 端到端路径走查 |

### v0.7.3 — Intake Router skill

**研发内容**
1. 新 skill `dev/.claude/skills/credit-analysis-router/SKILL.md`：
   - description 触发面：模糊/复合型信用分析需求（"帮我看看这家公司""这个组合有没有问题"）
   - 渐进式需求理解协议（四问，允许跳答/多答）：
     Q1 角色：您以什么身份决策？（M0 审贷/M1 投资/M2 承销/M3 交易/M4 风控/M5 融资/不确定→按问题特征推荐）
     Q2 对象：单发行人/债券组合/行业/全市场？
     Q3 深度：快速信号（L0）/决策快照（L1）/深度报告（L2）/专项？
     Q4 数据：仅公开数据（Mode A）/有外部数据源（Mode B 需用户显式提供）？
   - 输出《工作路径单》（yaml，字段对齐 registry schema）+ 推荐阅读顺序 + 质量门提醒
   - 不确定时的默认策略：信息不足→推荐 L0/L1 而非直接 L2；对象不明→先单发行人
2. `references/`：`work-paths.md`（从 registry 提取的路由视图，标注对应引擎版本）
